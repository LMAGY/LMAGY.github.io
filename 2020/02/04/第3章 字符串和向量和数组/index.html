第3章 字符串和向量和数组

命名空间

  1. 使用方法：using namespace::name ; 例如using std::cin
  2. 每个名字都需要独立的using声明
  3. 头文件中一般不用命名空间

标准库类型 string

  1. 拷贝初始化（用“=”），直接初始化不用“=”

  2. 直接初始化适用范围更广（毕竟面向对象）

    os<<s 写入输出流
    is>>s 读取字符流
    getline(is,s) 从is中读取一行赋给s
    s.empty（） s为空返回true，否则返回false
    s.size（） 返回s中字符的个数
    s[n] 检索第n个字符
    s1 + s2 返回连接后的结果
    s1 = s2 用s2的副本代替s1中原来的字符
    s1 == s2 判断是否相等
    s1 != s2 等性判断，大小写敏感
    <, <=, >, >= 字典序排序

  3. 读写string对象 ，cin只能读取非空白符，遇到空白结束

  4. 读取未知数量的string对象（commad + D是文件结束符）

  5. 使用getline读取一整行（可以保留空白符）

    getline(cin,line)从输入读取到line中，直到遇到换行符为止，换行符会被丢弃。

    getline会返回一个流对象，可以一直输入。

  6. string的empty和size操作

    string.empty( )会返回一个布尔值，string.size( )会返回字符串的长度

    这个长度值是一个string::size_type的类型(这是一个无符号的值)，所以千万不要和int混用。

  7. 字符串字面值不能直接相加

  8. 字符产不能直接相加不能直接相加，string对象可以加字面值，也可以加string对象。切记字符串字面值不是string对象

  9. 字典序，越往后越大

处理string对象中的字符

  1. 代码 含义
    isalnum(c) 当c是字母或数字时为真
    isalpha(c) 当c是字母时为真
    iscntrl(c) 当c是控制时为真
    isdigit(c) 当c是数字时为真
    isgraph(c) 当c不是空格但可打印时为真
    islower(c) 当c是小写字母时为真
    isprint(c) 当c是可打印字符时为真（空格或具有可视形式）
    ispunct(c) 当c是标点符号时为真(即c不是控制字符，数字，字母，可打印空白中的一种)
    isspace(c) 当c是空白时为真(即c是空格，横向制表符，纵向制表符，回车符，换行符，进纸符)
    isupper(c) 当c是大写字母时为真
    isxdigit(c) 当c是十六进制字母时为真
    tolower(c) toupper(c) 如果c是大写字母，输出对应的小写字母，否则原样输出c ; 如果c是小写字母，输出对应的大写字母，否则原样输出c。

    一般来说，C++应使用cname而非name.h的头文件。

处理每个字符使用基于范围的for语句

for (auto i : strings)

使用范围for语句改变字符串中的字符

  1. 关键是要用引用，for (auto& c : strings)

只处理一部分字符

  1. 使用下标或者使用迭代器，下标运算符[ ]接受的下标是size_type类型,返回值是对该位置字符的引用。s[0]到s[s.size()-1]
  1. c++不要求检测下标是否合法，使用超出范围的下标就会产生不可预知的结果

标准库类型vector(容器)

  1. vector表示对象的集合，其中所有对象的类型都相同，也被称为容器(container),vector其实是一个类模版。

  2. 模版本身不是类或函数，模版是为为编译器生成类或函数编写的一份说明。编译器根据模版创建类或函数的过程称为实例化，当使用模版时，需要指明编译器把类或函数实例化何种类型。对于类模版来说，我们通过提供额外的信息来指定模版到底实例化成什么样的类，需要提供哪些信息由模版决定。提供信息在模版后跟一对尖括号，在括号内加入信息。

    ivec 保存int类型的对象，类型是vector

  3. ( )是默认初识化的方法

  4. { }是列表初始化的方法，当其无意义时会考虑使用默认初识化的方法（即退化为（）

向vector对象中添加元素

  1. 使用vector的push_back可以向vector中加入元素
  2. 通常做法是定义一个空的vector，再用push_back( )往里面放，这样反而比给出vector的大小再往里面放的效率高（动态大于静态）
  3. 其他操作 .size( ) .empty( ) .push_back( ) 等
  4. 注意：不能使用下标给未定义的位置赋值或进行访问等操作，否则会造成缓冲区溢出这种类似的错误(buffer overflow)

迭代器(就像一个指针)

  1. 迭代器是一种通用机制来访问容器中的对象；所有标准库容器都能用迭代器访问，但是只有少部分支持下标访问。

  2. 类似指针类型，迭代器提供了对对象的间接访问。就迭代器而言，其对象是容器中的元素或者string对象中的字符。使用迭代器可以访问某个元素，迭代器也能从一个元素移动到另外一个元素。迭代器有有效和无效之分，和指针相似。有效的迭代器或者指向某个元素，或者指向容器中尾元素的下一位置；其他所有情况都属于无效

  4.     string v("hello,world!!!");
    auto b = v.begin();//b是一个迭代器，指向v的第一个元素
    auto e = v.end();//e是一个迭代器，指向v的尾元素的下一个位置，被称为尾迭代器/尾后迭代器
//若是容器为空则两者都返回尾后迭代器
<!--￼5-->

  5. 数组下标的类型是size_t

  6. 数组名有时候表示的表示指针指向首地址，有时候表示数组对象本身

  7. 尾后指针类型尾后迭代器，int * p = &a[n];

  8. 数组类型不是类类型,所以没有.begin( )和.end( )这种成员函数,用begin(a),begin(b)这种方式作用在上面

  9. 标准库类型的下标是size_type( )类型，无符号数；数组下标是有符号的数

重点在于：数组名不一定总是指针，大部分时间表示数组对象，当其作为指针时可以当作是一种退化，退化成类型迭代器的东西（但他是常量，只能赋值给其他指针，begin和end函数也能完成这种功能）

  1. 新旧风格代码混用时 ：不能使用string类型初始化即char * s = string类型对象是不合法的，char *s = string.c_str( )才可以
  2. 不能用标准库类型初始化数组，但是可以用数组初始化vector等容器

多维度数组

  1. 所谓多维数组就是数组的数组；
  2. 使用range for时除了最内层，其他控制变量都应该是引用类型

数组是一种类似字面值的东西，可以复制给string vector等对象（反之则不行）
