const 总结

  1. const对象必须初始化

  2. 默认状态下，const对象仅在文件内有效（无论是定义 还是声明，都应该在const前面加上extern，必须都要加上，少一个都不行；但是，对于普通的对象来说，只需要在使用的地方加上extern 就可以了，定义的地方并不需要）

  3. 非常量引用不能引用常量对象；常量引用能引用非常量对象（只是不能通过该引用改变这个非常量的值，但非常量对象的值是可以改变的）

  4. 非pointer to const不能指向常量；pointer to const能指向非常量对象（只是不能通过指针改变。。。自以为是认为自己指向了常量

  5. const pointer 表示指针本身是常量

  6. // C4227.cpp
// compile with: /W1 /c
int j = 0;
int &const i = j;   // C4227
//这个写法会给一个warning，但不会会被丢弃
<!--￼0-->

  7. 底层const的限制就不能忽略。拷贝时，拷入和拷出的对象必须具有相同的底层const资格，或者两个对象的数据类型必须能够转换。非常量可以转为常量，反之不行，具体来说就是 int * 类型能转化为const int * 类型，而const int * 不能转化为int *类型；int *可作为左值赋给cosnt int *，反之不行。

  8. 总结：对于指针来说顶层const可以忽略（拷贝时），对于引用来说，只存在底层const，对于const的对象也必须用const的引用const & ri
