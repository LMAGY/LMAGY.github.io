第6章 函数

函数基础

  1. 函数可以使用引用型形式参数，可以避免拷贝，同时可以返回额外信息，这叫做引用调用；直接传值的调用叫做传值调用

  2. 形参的初始化方式和和变量的初始化方式完全相同，需要注意顶层和底层const等性质

  3. 我们要尽量使用常量引用，因为可以将非常量作为实参传入形参，但是不能将常量作为实参传入形参定义为常量引用的函数，所以如果不需要在函数中改变该实参，则一律使用常量引用

  4. 数组形参不能直接用拷贝传入，传入的是指向数组首元素的指针；但是我们可以将形参写成类似数组的形式

  5. void print1(const int *a)
{
    cout<<a[2]<<endl;
}
void print2(const int a[])
{
    cout<<a[2]<<endl;
}

void print3(const int a[10]){
    cout<<a[2]<<endl;

}
//实际上类型都是const int*，数组的大小对函数的调用没有影响

函数重载

  1. 函数重载根据传入的实参调用同名的对应的函数，main函数不能够重载

  2. 对应重载的函数来说，它们应该在形参数量或形参类型上有所不同

  3. 只有返回值类型不同也是不行的

  4. 一个拥有顶层const的形参无法和另一个没有顶层const的形参区分开来

  5. 如果形参是某种类型的指针或引用，通过区分其指向的是常量对象还是非常量对象可以实现函数重载，此时const是底层（其指向的对象是常量，或者说他指向的对象不是常量，但不能通过其修改）

    当传递进去的是const对象时就只能用第二个lookup因为常量类型不能转化为非常量。相反，非常量可以转化为const，所以当传入非常量时上面两个函数都能被调用，这是编译器会优先调用非常量版本的函数,简而言之，const int 不行 const int *或者 const int & 就可以；

  6. 只有当函数的功能非常类似的时候再用函数重载

  7. const_cast有时和函数重载一起用比较合适

  8. 函数匹配又叫重载确定，指的是将函数调用与一组重载函数中的某一个关联起来

  9. 有多个函数可以匹配，但每一个都不是最佳选择。也将发生错误，被称为二义性调用

  10. 如果我们在内层作用域中声明名字，他将隐藏外层作用域中声明的同名实体。因此在不同的作用域中无法重载函数名。

  11. 变量名和函数名也不能相同，内层作用域的变量名会屏蔽外层的函数名

  12. 名字查找发生在类型检查之前

  13. 复习：const string &可以直接引用字符串字面值，string &则不可以

函数匹配

  1. 当几个重载函数的形参数量相等，或者某些形参的类型可由其它类型转换而来时，调用哪个重载函数将会变得困难，一般寻找形参类型与实参类型最匹配的可行函数

  2. 在含有多个形参匹配时，可能存在二义性拒绝请求的情况：比如，匹配第一个函数，第一个位置需要类型转换，匹配第二个函数，第二个位置需要，编译器无法确定哪个是最佳匹配，从而无法匹配

  3. 当存在类型提升时一般会直接提升为int，比如char可能会直接提升为int匹配int形参，而不是匹配short形参

  4. 小数字面值是double，能转化为long和float，存在二义性

  5. 不能把普通引用绑定到const对象上，函数重载形参不能是顶层const，因为常量可以给非常量传参

第4章 表达式

基础

  1. 运算符作用于运算对象(变量或者字面值)即为表达式（复合表达式）
  2. 函数调用是一种特殊的运算符
  3. 对象被作为右值时，用的是对象的值(内容)，用作左值时，用的是对象的身份(在内存中的位置)
  4. decltype( * p) 推断出来的是int &类型,* p是一个左值，表示一个对象，因此是引用类型
  5. 优先级和结合律和求职顺序决定了复合表达式
  6. 当求值顺序未定义时，当复合表达式指向并修改了同一个对象时会产生不可预知的行为
  1. 算数运算符运算的结果都是右值(有的运算符运算的结果不一定是右值)
  2. 算数运算符能提升运算对象，使得表达式的类型统一
  1. c++规定(-m/n) = (m/-n) = -(m/n) , (-m)%n = -(m%n), m%(-n) = m%n
  2. 赋值运算满足右结合律
  3. 退化表示对象变成首元素地址（发生在数组中），提升表示一种类型的转换，表达式中可能会出现（算数运算或者逻辑运算包括关系运算），退化和提升没有啥关系
  4. 逻辑运算符和关系运算符返回的结果是布尔类型
  5. while（i==ture）会出错，true会提升为int的1，只有当i为1时才返回true（关系运算符的结果是bool类型）
  6. 赋值运算符不会产生提升，反而会进行类型转化 ，int a ； a = 3.13；3.13会转化为int类型 的
  7. 有的迭代器不支持算数运算，自增自减可以，建议使用前置版本的自增自减运算
  8. 后置版本的自增自减是先进行自增自减，然后返回复制的之前的值
  9. 不要在一个表达式中多次出现会被改变的运算对象
  10. 后置自增自减运算符的优先级低于 .运算符(a.size())的优先级，后者高于*（解引用）运算符
  11. ci是迭代器*ci++很常用，先算ci++返回ci再解引用

条件运算符

  1. ? a:b 返回结果既可以当作左值也可以当作右值（于a和b是左值还是右值有关）
  2. 条件表达式可以嵌套，满足右结合律，应该从右向左读
  3. 条件运算符的优先级非常非常低，所以应该将其用括号括起来

位运算符

  1. 关于符号位如何处理没有明确的规定，所以建议仅仅将位运算符用于处理无符号类型
  2. 左移右移时，短的整型(比如char等)可能会提升为int

sizeof运算符

  1. 他可以返回表达式结果类型的大小
  2. sizeof右侧的元素其实不会被计算
  3. sizeof是运算符 右结合律，从右往左看(不加括号也行)

逗号运算符

  1. 对于逗号运算符来说，首先对左侧的表达式求值，然后丢弃；真正的结果是右侧表达式的值
  2. 一般在for循环中使用

类型转换

  1. 隐式转换自动发生，一般朝不损失精度的方向转换

  2. 初始化和赋值时，右侧对象转换为左侧对象的类型

  3. 算数转换：当无符号类型和有符号类型相加时，谁的类型大转化为谁的（从int转化为unsigned可能会发生错误)

  4. 表达式中比int类型小的一般会先转化为int类型

  5. cast-name(expression) 为命名式的强制转换

  6. static_cast是普通的类型转换，当损失精度时不报错

  7. const_cast只能改变对象的的底层const (指针指向常量)，不能改变对象的类型

  1. char a = 'a';
const char *pc = &a;
char *p = const_cast<char*> (pc);//pc是底层const，转变为char *，但是通过p修改a的做法是未定义的
*p = 'b';
//*pc = 'c'; 不合法无法通过底层const修改a
cout<< a <<endl;
a = 'c';
cout<<a<<endl;
    1. reinterpret_const在位的层面上改变对象的类型
    2. 除了底层const的指针，可以把任何类型的指针的值赋给void *类型的指针

标准库类型vector(容器)

  2. 模版本身不是类或函数，模版是为为编译器生成类或函数编写的一份说明。编译器根据模版创建类或函数的过程称为实例化，当使用模版时，需要指明编译器把类或函数实例化何种类型。对于类模版来说，我们通过提供额外的信息来指定模版到底实例化成什么样的类，需要提供哪些信息由模版决定。提供信息在模版后跟一对尖括号，在括号内加入信息。

    ivec 保存int类型的对象，类型是vector

第2章 auto和decltype总结

auto

  1. auto定义的变量必须有初始值

  2. auto也能在一条声明语句中声明多个变量，但一条声明语句只有一个基本数据类型，所以该语句中所有变量的初始基本数据类型*必须相同 *

  3. 编译器会以 *被引用变量 *的值类型作为auto的类型

  4. auto会忽略顶层const，同时保留底层const。如果希望 推断出来的auto是一个顶层const，可以用 const auto f = j

decltype

  1. 作用：选择并返回操作数的数据类型。编译器分析表达式的类型，却不实际计算表达式的值。

  2. decltype 处理顶层const和引用的方式不同。decltype会返回变量真实的类型（包括顶层const和引用在内）

  3. 引用从来都是作为其所指对象的同义词出现，只有用在decltype处是一个例外

  4. 有些表达式能向decltype返回引用类型。

  5. decltype 的表达式如果是加上了括号的变量，结果是引用。

第2章 const总结

const 总结

  1. const对象必须初始化

  2. 默认状态下，const对象仅在文件内有效（无论是定义 还是声明，都应该在const前面加上extern，必须都要加上，少一个都不行；但是，对于普通的对象来说，只需要在使用的地方加上extern 就可以了，定义的地方并不需要）

  3. 非常量引用不能引用常量对象；常量引用能引用非常量对象（只是不能通过该引用改变这个非常量的值，但非常量对象的值是可以改变的）

  4. 非pointer to const不能指向常量；pointer to const能指向非常量对象（只是不能通过指针改变。。。自以为是认为自己指向了常量

  5. const pointer 表示指针本身是常量

  6. // C4227.cpp
// compile with: /W1 /c
int j = 0;
int &const i = j;   // C4227
//这个写法会给一个warning，但不会会被丢弃

  7. 底层const的限制就不能忽略。拷贝时，拷入和拷出的对象必须具有相同的底层const资格，或者两个对象的数据类型必须能够转换。非常量可以转为常量，反之不行，具体来说就是 int * 类型能转化为const int * 类型，而const int * 不能转化为int *类型；int *可作为左值赋给cosnt int *，反之不行。

  8. 总结：对于指针来说顶层const可以忽略（拷贝时），对于引用来说，只存在底层const，对于const的对象也必须用const的引用const & ri
